<h1 align="center">fOboromi</h1>
<h4 align="center">a WIP ounce emulator foundation written in Rust</h4>

## Overview

**fOboromi** is a work-in-progress fork of oboromi targeting the Nintendo Switch 2.

It currently focuses on building a low-level emulator foundation, including CPU emulation, storage emulation, firmware boot groundwork, and early GPU infrastructure.

> Experimental project.  
> Commercial games and full firmware are not runnable yet.

## Current Features

- 8-core ARMv8 CPU emulation
- 12GB shared memory model
- SM86 (NVIDIA Ampere) GPU stub / groundwork
- Sparse NAND image support
- Raw `package2` kernel loading
- eframe-based debugging GUI
- CPU test runner
- Boot phase tracking and status reporting

## Firmware Support

The emulator currently supports or includes groundwork for:

- `nand.img` storage loading
- raw `package2` boot input
- `prod.keys` parsing
- NCA header parsing
- firmware directory scanning
- key area handling for firmware-related content

The current direction is to support firmware layouts like: `nand.img`, `package2`, `firmware/prod.keys`, and extracted `*.nca` files. Full NCA content decryption and execution are still in progress.

## Project Structure

- **core**: core library
  - **audio**
  - **fs**: filesystem and storage layer
    - VFS implementation
    - NAND / storage emulation
    - `keys.rs`: `prod.keys` parser
    - `nca.rs`: NCA header parsing / decryption
    - `pfs0.rs`: partition filesystem reader
    - `firmware.rs`: firmware loader / scanner
  - **gpu**: Ampere SM86 emulation / recompilation
    - SASS to SPIR-V conversion
    - Vulkan translation groundwork
    - constants and register mapping
  - **nn**: HLE firmware emulation
  - **sys**: full emulation state and boot pipeline
  - **tests**

- **docs**
  - `sm86.pdf`: autogenerated SM86 instruction set document

- **gui**: eframe-based debugging UI

- **scripts**: helper tools / autogenerators
  - `decoder_generator.rs`: generates parser logic from SM86 descriptions
  - `sm_86_instructions.txt`: instruction dump sourced from [DocumentSASS](https://github.com/0xD0GF00D/DocumentSASS)
  - `sm_86_latencies.txt`: instruction latency data sourced from [DocumentSASS](https://github.com/0xD0GF00D/DocumentSASS)

## Limitations
- No commercial game support
- No complete firmware boot yet
- No full NCA content execution yet
- GPU implementation is still an early stub / foundation
## Roadmap
- full NCA decryption and section extraction
- ExeFS / NSO parsing and loading
- kernel boot execution improvements
- expanded HLE system services
- deeper GPU command and shader handling
